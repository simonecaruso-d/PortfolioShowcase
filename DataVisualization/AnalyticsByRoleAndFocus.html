01 · Problem

Fragmented sources, no shared view

Revenue lived in Oracle, pipeline in HubSpot, headcount in SharePoint. Finance, Sales, and Delivery each extracted their own slice and reconciled by hand: no single session ever showed the full picture with no focus on what each role actually needed

Before

  • Oracle ERP, PostgreSQL DB, HubSpot, and SharePoint data extracted separately; no live connection between them
  • Treasury KPIs recalculated manually in Excel each cycle by the finance team
  • Sales pipeline and project delivery siloed: no end-to-end view from HubSpot opportunity to invoiced margin
  • Team performance visible only at month-end close, not continuously: no intra-period course correction

After

  • Single Power BI model fed live from 5 sources: Oracle ERP, PostgreSQL DB, HubSpot REST API, SharePoint Excel, embedded JSON
  • Treasury KPIs encoded in DAX with cascading dependencies: no manual exports, always in sync
  • End-to-end visibility: from HubSpot deal stage through project execution to invoice collection and cash
  • Role-based views (Team Officer · Client Account · Project Manager) segmented by domain: each role sees exactly what it needs

Power Automate Historic Capital Working Days - Data Warehousing pipeline
The problem

Capital Working Days is a point-in-time metric: every day the ERP overwrites the underlying receivables and ARNI balances, so each refresh gives a current value but history is permanently lost. There was no way to plot trends, set alerts on drift, or do period-over-period comparison beyond what the fact tables natively preserved

The solution

A scheduled Power Automate flow runs once a day, reads the current CWD value exposed by the refreshed report, and appends a timestamped row to a SharePoint list, building a lightweight data warehouse one row per day. Power BI then loads this history table as a dedicated query, enabling true time-series analysis of working-capital drift without touching the source ERP

04 · DAX

The logic, in code

200+ measures across working capital, revenue, pipeline, engagement, project tracking, and more

Margin % Production PY measure
Production margin % for the prior calendar year. The ratio stays frozen on the closed year regardless of the current period slicer
 1VAR PreviousYear  = YEAR(TODAY()) - 1
 2VAR Revenue       = CALCULATE([Revenue Actual Production], YEAR('Date Table'[Date]) = PreviousYear)
 3VAR Cost          = CALCULATE([Cost Actual Production], YEAR('Date Table'[Date]) = PreviousYear)
 4VAR Margin        = Revenue - Cost
 5RETURN IF(Revenue IN {0, BLANK()},
 6        IF(Margin IN {0, BLANK()}, 0, IF(Margin > 0, 1, -1)),
 7        Margin / Revenue)
eNPS Storytelling measure
Text icon comparing the two most recent eNPS survey results: returns "✓" if the score held or improved, "⊘" if it declined, blank if no data. Displayed as a directional signal alongside the raw eNPS score in the Engagement view
 1VAR LatestDate       = MAX('eNPS'[Date])
 2VAR SecondToLastDate = CALCULATE(MAX('eNPS'[Date]), 'eNPS'[Date] < LatestDate)
 3VAR After            = CALCULATE([eNPS], 'eNPS'[Date] = LatestDate)
 4VAR Before           = CALCULATE([eNPS], 'eNPS'[Date] = SecondToLastDate)
 5RETURN IF(After = BLANK(), " ", IF(After < Before, "⊘", "✓"))
Cashed measure
Total cash collected per project: sum of VAT-included amounts for invoices with Paid or Closed status, limited to FTA, NCA, and SA invoice types. Direct cash-in KPI used in the PM Portfolio journey view at the Cashing stage
 1CALCULATE(SUM('Aperti+IncPag'[VAT Included Amount]),
 2    'Aperti+IncPag'[Invoice Expiration Status] IN {"Paid", "Closed"} && 'Aperti+IncPag'[Invoice Type] IN {"FTA", "NCA", "SA"})
ARNI Cluster column · Activity Detail
Classifies unbilled work into four ageing buckets combining recency and invoice overdue status
 1VAR Maturazione = DATEDIFF('Activity Detail'[Execution Planned Date], TODAY(), DAY)
 2VAR Scadenza    = DATEDIFF('Activity Detail'[Invoice Planned Date], TODAY(), DAY)
 3RETURN
 4    IF(Maturazione <= 60,
 5        IF(Scadenza > 0, "Corrente-Scaduto",    "Corrente-Non scaduto"),
 6        IF(Scadenza > 0, "Non corrente-Scaduto", "Non corrente-Non scaduto"))
