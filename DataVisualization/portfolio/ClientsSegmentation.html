01 · Problem

Each market was running its own spreadsheet

Territory managers had to export raw sales, apply local rules in Excel, and reconcile by hand : every cycle.
Inconsistent definitions, no shared time windows, no audit trail

Before

  • Four separate spreadsheets, one per market, each with its own rating logic
  • Time windows defined per analyst; YoY and PoP comparisons recomputed manually each cycle
  • No way to compare strategic action across territories: every output was a snapshot, not a system
  • Analysis performed on specific products only

After

  • Single model sourced live from Azure Analysis Services: zero local data
  • Dynamic time intelligence windows with switchable YoY and PoP growth
  • Cascading classification (Usage → Lifecycle → Action) encoded in DAX, governed centrally
  • Market and Product-specific overrides handled by parameter tables, not by forking the report

04 · DAX

The logic, in code

46+ measures across segmentation, time intelligence, growth analytics, and dynamic formatting.
The standout is the cascading classification: Class 3 reads Class 2 reads Class 1,
each evaluated at row context with market-specific overrides

Class 1 - Usage Level measure
Benchmarks each customer against the average sales for their rating group in the selected period. Returns High User (at or above average), Low User (below average but with historical sales), No User (zero sales in period), or No Data (no historical sales, or Greece C-rating exclusion applies)
DAX
 1VAR Country      = SELECTEDVALUE('DDP Data'[Market Code])
 2VAR Rating       = SELECTEDVALUE(Customer[Customer Rating])
 3VAR TouchpointId = SELECTEDVALUE(Customer[Customer ID])
 4
 5VAR Output = IF([Historical Sales] IN {0, BLANK()},
 6                "No Data",
 7                IF([Period Sales] IN {0, BLANK()},
 8                    "No User",
 9                    IF([Period Sales] >= [Period Sales | Average by Rating], "High User", "Low User")))
10
11RETURN SWITCH(TRUE(),
12    Country = "GRC" && LEFT(Rating, 1) = "C" && NOT TouchpointId IN VALUES('Greece CA Customers To Keep'[Customer Id]), "No Data",
13    Output)
Class 2 - Lifecycle Stage measure
Places each customer on a lifecycle arc by comparing sales across three consecutive time windows and two sub-period halves. The 11-branch Switch resolves to: Loyal · Potentially Loyal · Back · New · Lost · Not Buying · Never Using · No Data
DAX
 1VAR Country      = SELECTEDVALUE('DDP Data'[Market Code])
 2VAR Rating       = SELECTEDVALUE(Customer[Customer Rating])
 3VAR TouchpointId = SELECTEDVALUE(Customer[Customer ID])
 4
 5VAR ThisPeriod         = 0 + [Period Sales]
 6VAR LastPeriod         = 0 + [Last Period Sales]
 7VAR SecondToLastPeriod = 0 + [Second To Last Period Sales]
 8VAR CurrentSubPeriod   = 0 + [Current SubPeriod Sales]
 9VAR LastSubPeriod      = 0 + [Last SubPeriod Sales]
10
11VAR Output = SWITCH(TRUE(),
12    [Historical Sales] IN {0, BLANK()}, "No Data",
13    SecondToLastPeriod = 0 && LastPeriod = 0 && ThisPeriod = 0, "Never Using",
14    SecondToLastPeriod = 0 && LastPeriod = 0 && ThisPeriod <> 0, "New",
15    LastPeriod <> 0 && ThisPeriod = 0, "Not Buying",
16    SecondToLastPeriod = 0 && LastPeriod <> 0 && ThisPeriod <> 0, "Potentially Loyal",
17    SecondToLastPeriod <> 0 && LastPeriod = 0 && ThisPeriod = 0, "Lost",
18    SecondToLastPeriod <> 0 && LastPeriod = 0 && ThisPeriod <> 0, "Back",
19    SecondToLastPeriod <> 0 && LastPeriod <> 0 && ThisPeriod = 0, "Not Buying",
20    LastPeriod <> 0 && CurrentSubPeriod <> 0 && LastSubPeriod <> 0, "Loyal",
21    LastPeriod <> 0 && OR(CurrentSubPeriod <> 0, LastSubPeriod <> 0), "Potentially Loyal",
22    SecondToLastPeriod <> 0 && LastPeriod <> 0 && ThisPeriod <> 0, "Potentially Loyal")
23
24RETURN SWITCH(TRUE(),
25    Country = "GRC" && LEFT(Rating, 1) = "C" && NOT TouchpointId IN VALUES('Greece CA Customers To Keep'[Customer Id]), "No Data",
26    Output)
Class 3 - Strategic Action measure
Reads Class 1 and Class 2 and emits one of five actions: Develop · Accelerate · Elevate · Reconnect · Acquire. Low User + Not Buying also maps to Reconnect (win-back). Falls back to the Class 1 label when no matrix condition matches, preserving full context for edge cases
DAX
 1VAR ClassOne = [Class 1]
 2VAR ClassTwo = [Class 2]
 3
 4RETURN SWITCH(TRUE(),
 5    ClassOne = "No Data", "No Data",
 6
 7    ClassOne = "No User" && ClassTwo = "Never Using", "Acquire",
 8    ClassOne = "No User" && ClassTwo IN {"Not Buying", "Lost"}, "Reconnect",
 9
10    ClassOne = "Low User" && ClassTwo = "Not Buying", "Reconnect",
11    ClassOne = "Low User" && ClassTwo IN {"Potentially Loyal", "Back", "New"}, "Elevate",
12    ClassOne = "Low User" && ClassTwo IN {"Loyal"}, "Accelerate",
13
14    ClassOne = "High User" && ClassTwo IN {"Loyal", "Potentially Loyal", "Back", "New"}, "Develop",
15
16    ClassOne)
Greece CA To Exclude column · Customer
Flags Greek customers with a C-rating who are absent from the approved whitelist. Returns 1 (exclude from C-rating analytics) or 0 (include). All non-GRC rows return 0. Computed at load time in the Customer table; referenced in Class 1 and Class 2 for the market-override gate
DAX
 1VAR MarketCode = Customer[Market Code]
 2VAR Rating     = LEFT(Customer[Customer Rating], 1)
 3VAR CustomerId = Customer[Customer ID]
 4VAR ToKeep     = VALUES('Greece CA Customers To Keep'[Customer Id])
 5
 6RETURN SWITCH(TRUE(), MarketCode = "GRC" && Rating = "C", IF(CustomerId IN ToKeep, 0, 1), 0)
Portugal ABM To Exclude column · DDP Data
Flags Portuguese rows where the Area Business Manager is blank: a data-quality signal from the SalesForce join indicating unassigned territory. Rows returning 1 are excluded from territory-level analytics. Computed at load time in the DDP Data fact table
DAX
 1VAR MarketCode = 'DDP Data'[Market Code]
 2VAR ABM        = 'DDP Data'[Area Business Manager]
 3
 4RETURN SWITCH(TRUE(), MarketCode = "POR" && ABM = BLANK(), 1, 0)
Period Sales measure
Core parametric measure: sums Sales Amount or Quantity for a sliding window anchored to the date slicer. Window width is driven by the Time Parameter (1 M · 3 M · 6 M · 12 M · YTD); measurement unit by the Dimension Parameter (Value · Quantity). Every downstream classification chains off this single measure
DAX
 1VAR ThresholdDate        = SELECTEDVALUE('Date Table'[Date])
 2VAR MeasurementParameter = SELECTEDVALUE('Dimension Parameter'[Text])
 3VAR TimeParameter        = SELECTEDVALUE('Time Parameter'[Text])
 4
 5VAR StartDate = SWITCH(TimeParameter,
 6                    "12 Months",   ThresholdDate - 367,
 7                    "6 Months",    ThresholdDate - 182,
 8                    "3 Months",    ThresholdDate - 92,
 9                    "1 Month",     ThresholdDate - 32,
10                    "Year To Date", DATE(YEAR(ThresholdDate), 1, 1))
11
12VAR ValueOutput    = CALCULATE(SUM('DDP Data'[Sales Amount]),   'DDP Data'[Date] <= ThresholdDate && 'DDP Data'[Date] >= StartDate)
13VAR QuantityOutput = CALCULATE(SUM('DDP Data'[Sales Quantity]), 'DDP Data'[Date] <= ThresholdDate && 'DDP Data'[Date] >= StartDate)
14
15RETURN SWITCH(MeasurementParameter, "Quantity", QuantityOutput, "Value", ValueOutput)
Historical Sales measure
Anchored baseline: sums sales from a fixed start date (Dec 2023) through the threshold date, with Brand filter removed. Used by Class 1 to decide whether a customer has any history at all: the gate that separates No Data from active classification. Brand filters are stripped so the check is customer-level, not SKU-level
DAX
 1VAR ThresholdDate        = SELECTEDVALUE('Date Table'[Date])
 2VAR MeasurementParameter = SELECTEDVALUE('Dimension Parameter'[Text])
 3VAR HistoricalDateStart  = DATE(2023, 12, 1)
 4
 5VAR ValueOutput    = CALCULATE(SUM('DDP Data'[Sales Amount]),
 6                        'DDP Data'[Date] <= ThresholdDate && 'DDP Data'[Date] >= HistoricalDateStart,
 7                        REMOVEFILTERS('DDP Data'[Brand]))
 8VAR QuantityOutput = CALCULATE(SUM('DDP Data'[Sales Quantity]),
 9                        'DDP Data'[Date] <= ThresholdDate && 'DDP Data'[Date] >= HistoricalDateStart,
10                        REMOVEFILTERS('DDP Data'[Brand]))
11
12RETURN SWITCH(MeasurementParameter, "Quantity", QuantityOutput, "Value", ValueOutput)
Current vs Last Year Sales measure
Percentage change between Period Sales and Same Period Last Year Sales. Returns Blank when the prior-year value is zero or blank, avoiding division noise. Formatted as a percentage; powers the YoY growth KPI card and the Positive/Negative growth customer counts
DAX
 1VAR CPS       = [Period Sales]
 2VAR SPLY      = [Same Period Last Year Sales]
 3VAR PreOutput = (CPS - SPLY) / SPLY
 4
 5RETURN IF(NOT SPLY IN {0, BLANK()}, PreOutput)
Current vs Last Period Sales measure
Percentage change between Period Sales and Last Period Sales (period-over-period). Same guard against zero denominators. Together with the YoY variant, these two form the inputs to the Overall Growth switcher
DAX
 1VAR CPS       = [Period Sales]
 2VAR LPS       = [Last Period Sales]
 3VAR PreOutput = (CPS - LPS) / LPS
 4
 5RETURN IF(NOT LPS IN {0, BLANK()}, PreOutput)
Overall Growth measure
Single switchable growth percentage driven by the Time Comparison Parameter slicer: routes to period-over-period or same-period-prior-year in one measure. This is the only growth figure exposed in KPI cards: no separate YoY / PoP measures required on the canvas
DAX
 1VAR CompareParameter = SELECTEDVALUE('Time Comparison Parameter'[Compare With])
 2VAR SPPY             = [Current vs Last Year Sales | Numerical]
 3VAR PPSY             = [Current vs Last Period Sales | Numerical]
 4
 5RETURN SWITCH(CompareParameter, "Previous Period Same Year", PPSY, "Same Period Previous Year", SPPY)
Format String Definition format string
Applied as a dynamic format string expression on every sales measure. Reads the Dimension Parameter slicer and returns the correct number format at render time: integer thousands-separator for Quantity, Euro currency with two decimals for Value. One expression governs all KPI cards: no per-visual overrides needed
DAX
 1VAR Threshold   = SELECTEDVALUE('Date Table'[Date])
 2VAR Measurement = SELECTEDVALUE('Dimension Parameter'[Text])
 3
 4RETURN SWITCH(Measurement, "Quantity", "#,##0", "€ #,##0.00")
