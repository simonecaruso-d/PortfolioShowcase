Assie

A five-agent pipeline that reads Italian public tender documents and rewrites candidate CVs to match

01 - Problem

CV adaptation at scale is manual and error-prone

Italian public tenders require tailored CV submissions per candidate per lot. Doing this by hand is slow, inconsistent, and risks inventing data that isn't in the original CV

Without Assie

  • |Each tender requires hours of manual reading to extract what's actually required per candidate profile
  • |CV adaptation is done by hand: inconsistent across candidates, error-prone under deadline pressure
  • |Risk of embellishing or inventing data
  • |Post-submission feedback loops are slow: edits require re-reading the full tender again
  • |No standard output format: every bid team formats differently

With Assie

  • →Tender synthesized in one click: key content, activities and scope
  • →Per-candidate CV drafting guided by structured requirement interpretation from the Instructor agent
  • →Seller agent adapts tone and emphasis to bid strategy, without adding data not in the original CV
  • →Conversational Reviewer loop: type feedback, get an updated CV instantly
  • →One-click PDF export of the final reviewed output

Core constraint driving the design: the system must never invent data. Every agent's system prompt explicitly prohibits adding information not present in the source CV. The Drafter populates from the CV; the Seller reframes emphasis. This is enforced in prompt, not code, which means it's auditable and adjustable without changing architecture

Five agents, one pipeline

Model selection rationale: each agent uses the model best suited to its task. DeepSeek-R1's reasoning depth handles synthesis and review. Qwen3-30B's instruction-following precision handles requirement interpretation. Llama-4 Maverick's speed handles high-volume drafting. All routed through OpenRouter: swapping models or changing settings like the temperature requires changing one line.

03 - Workflow

Tender in, tailored CVs out

The pipeline runs in two phases: a one-time tender synthesis, then per-candidate profile generation followed by an open-ended conversational review loop.

01
Upload
Tender + CV inputs
  • Upload one or more tender PDFs
  • Enter candidate names (comma-separated)
  • Enter role dictionary per candidate
  • CVs auto-located from directory
02
Synthesize
Tender compression
  • MarkItDown parses tender PDFs
  • Synthetizer strips boilerplate
  • Output: compact scope summary
  • Used as context for all downstream agents
03
Draft
Per-candidate generation
  • Instructor interprets requirement per person
  • MarkItDown parses each CV (PDF/DOC/DOCX)
  • Drafter populates required fields from CV
  • Seller adapts to tender + role dictionary
04
Review
Conversational refinement
  • Profiles displayed in Markdown
  • User types specific feedback
  • Reviewer applies minimal targeted edits
  • Full chat history maintained as context
05
Export
PDF generation
  • Final output from chat or generation
  • fpdf renders to downloadable PDF
  • ASCII-safe: non-standard chars stripped
  • One click from Gradio interface
04 - Product Design

Built for non-technical users under deadline

The Gradio interface with glassmorphism styling was deliberate: bid teams are not developers. The UI needs to feel approachable

Conversational Review

Chat-Driven Editing

After initial generation, the user interacts with the Reviewer via a chatbot interface. The full conversation history is passed as context so the agent can track and reconcile multiple rounds of feedback.

Gradio Chatbot component with copy button
Both Chatbot Enter and Button submit supported
Reset button clears history without clearing generation
