Wholesaler Data Upload

A Streamlit data validation tool built for pharmaceutical wholesalers to upload, validate, and submit monthly sales data, without manual intervention and with a three-layer compliance engine

Wholesalers submit Excel files in inconsistent formats. This app enforces a strict schema: 13 required columns with typed validation, Italian month name normalization, and quality checks, before data reaches the database

Streamlit Pandas Data Validation
01 - Problem

Wholesaler data arrives inconsistent by default

Pharmacies and wholesalers submit monthly sales files in varied formats: wrong column names, mixed data types, Italian month names, negative quantities, and out-of-range years. Manual review was the bottleneck

Without the tool
  • |Wholesalers use inconsistent column names and order
  • |Month submitted as "marzo" or "3" unpredictably
  • |Quantities sometimes negative, years sometimes invalid
  • |Null values in required fields reach the database silently
  • |Manual review needed before every import
With the tool
  • →13-column schema enforced on upload, missing columns flagged instantly
  • →Italian month names auto-converted to integers, case-insensitive
  • →Negative quantities and out-of-range years caught in quality layer
  • →Null counts shown per column before submission
  • →Submit button enabled only after all checks pass

Key design principle: validation is a gate, not a warning. The submit button is never shown until all three layers pass. Quality warnings are surfaced as warnings, not blockers: the wholesaler can proceed but sees exactly what's imperfect before committing

03 - Validation

Three layers, early exit on failure

RunComplianceChecks orchestrates all three layers in sequence. If any layer fails, it returns immediately

01
Column Validation
Schema presence check
  • ValidateColumns checks all 13 required columns
  • Missing columns → immediate failure + list
  • Extra columns flagged as warning (not blocking)
  • Early exit if any column missing
02
Type Coercion
Month normalization + type enforcement
  • ConvertMonthToNumber: Italian names → int
  • Case-insensitive: "Marzo" = "marzo" = 3
  • ValidateDataTypes: int64 / float64 / object per column
  • Any coercion failure → error list + exit
03
Quality Checks
Business rule validation
  • Null count per required column
  • Quantity < 0 flagged
  • Year outside [2000, current year] flagged
  • Month outside [1, 12] flagged
  • Duplicate rows detected and counted
